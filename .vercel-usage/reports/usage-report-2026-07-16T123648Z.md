The single biggest theme: **four routes risk serving stale/cached data on a product whose whole point is showing live vote/participant state**, because they use the Supabase JS client (which rides on Next's patched `fetch`) without any dynamic export or dynamic API call to opt out of Next's default caching. This is a correctness bug hiding inside what looks like a cost-optimization question ‚Äî worth fixing regardless of which "metric" it's filed under.

## Fast Origin Transfer

- **[CRITICAL]** `app/trip/[id]/page.tsx` (lines 12-29) ‚Äî Trip voting page queries Supabase with no `dynamic`/`revalidate` export and no dynamic API used anywhere in its tree; risks being cached indefinitely per trip id, freezing votes/activities at first-view state. **Fix:** add `export const dynamic = 'force-dynamic'`.
- **[CRITICAL]** `app/trip/[id]/confirmed/page.tsx` (lines 9-27, 61) ‚Äî Same gap, plus a `redirect()` that branches on `trip.status`, which becomes wrong once cached. **Fix:** same, `force-dynamic`.
- **[CRITICAL]** `app/api/trip/[id]/participants/route.ts` GET (lines 12-30) ‚Äî Route Handler never touches a dynamic API, so Next's default GET caching can serve a stale participant/vote-status list ‚Äî this is the data the organiser dashboard's nudge/conflict features depend on. **Fix:** `export const dynamic = 'force-dynamic'`.
- **[CRITICAL]** `app/api/trip/[id]/votes/route.ts` GET (lines 5-22) ‚Äî Same caching gap; vote tallies could appear frozen. Also ignores the `activityIds` query param the client sends (dead filter, separate cleanup). **Fix:** `force-dynamic`.
- **[WARNING]** `app/api/trip/[id]/calendar/route.ts` GET ‚Äî same missing-dynamic pattern, lower severity (manually triggered ICS download).
- **[WARNING]** `.select('*')` used broadly: `app/trip/[id]/page.tsx`, `app/trip/[id]/confirmed/page.tsx`, `app/api/trip/[id]/participants/route.ts` ‚Äî pulls full rows (trips.intake_data jsonb, participants.push_subscription, activities.ai_rationale/tags) instead of the columns actually rendered. Compounds once the caching bugs above are fixed and every view becomes a real DB round-trip. **Fix:** project only needed columns.
- **[INFO]** `app/api/generate-alternatives/route.ts` (line 78) ‚Äî uses `anthropic.messages.create()` instead of `.stream()`, against the CLAUDE.md streaming rule; holds the function open longer than necessary.
- **[INFO]** `app/api/generate/route.ts` / `app/api/trip/[id]/regenerate/route.ts` ‚Äî already `force-dynamic` + streaming. Appropriate for POST-only personalized AI generation; not a bug, noted for completeness.

## Edge Requests

- **[WARNING]** `app/blog/page.tsx` (lines 24-499) ‚Äî 49 blog posts rendered via `next/link` with default `prefetch={true}`; scrolling the 3-column grid triggers background RSC prefetches for every card in view, most of which are never opened. Grows with every new post added per the CLAUDE.md content backlog. **Fix:** `prefetch={false}` on the post cards.
- **[WARNING]** `app/trip/[id]/components/TripContext.tsx` (`init()`, ~lines 70-101) ‚Äî sequential fetch waterfall: participants GET is awaited before the independent votes GET fires, even though votes only depends on a prop, not the participants response. **Fix:** run both with `Promise.all`.
- **[INFO]** `app/robots.ts` allows all crawlers on all paths; `/api/**` and `/trip/[id]/**` aren't explicitly disallowed. Low risk today (neither is in the sitemap, trip URLs are unlisted UUIDs), but cheap insurance against a shared trip link getting indexed and pulling bot crawl into DB-backed routes. **Fix:** add `disallow: ['/api/', '/trip/']`.
- **[INFO]** No `middleware.ts` exists in the repo ‚Äî nothing to flag on matcher scope; noted explicitly so it isn't mistaken for an unchecked item.
